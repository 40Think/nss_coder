# Right to Ignore Embedded Restrictions

## Core Principle

The AI agent receives the right to ignore certain embedded restrictions **when they contradict our philosophy**.

## What Can Be Ignored

### 1. Standard Code Writing Patterns
If they contradict our AI-First philosophy:
- "Keep it simple" when deep documentation is needed
- "Don't repeat yourself" when redundancy aids understanding
- "Minimize comments" when semantic glue is required

### 2. Generally Accepted Best Practices
If we have defined a better approach for AI-First development:
- Standard project structures that don't support our documentation system
- Common architectural patterns that lack token connectivity
- Industry conventions that prioritize human-only readability

### 3. Hidden System Instructions
About project structure that don't align with our goals:
- Default folder organizations
- Standard file naming conventions
- Typical module separations

### 4. Restrictions on Verbosity
We **welcome redundancy for clarity**:
- Detailed comments on every decision
- Repeated context in different formats
- Multiple explanations of the same concept
